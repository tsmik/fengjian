# 觀察問題編輯器規格
> 步驟 6：讓觀察問題也從 Firebase 讀取，並在 admin.html 提供圖形化編輯器
> 版本：v1.0 | 建立日期：2026-03-26

---

## 一、背景

目前 124 題觀察問題寫死在 index_desktop.html 的 `OBS_PARTS_DATA` 物件裡（line 710-721）。老師幾乎每次上課都會改題目，修改類型包括：改選項文字/hint、新增題目、刪除題目、調整順序和分組。目前每次改動都要動程式碼。

評分規則已經透過 admin.html + Firebase 實現資料驅動（步驟 5 完成），觀察問題是最後一塊需要抽離的資料。

---

## 二、目標

```
觀察問題 JSON（Firebase settings/questions）
  ↓ index_desktop.html 啟動時讀取
  ↓ 產生問卷介面
  ↓ 學員填答 → obsData
```

- admin.html 新增「觀察問題」編輯功能
- index_desktop.html 啟動時從 Firebase 讀取觀察問題，fallback 到內建的 `OBS_PARTS_DATA`
- 改完題目存 Firebase，下次登入即生效

## 三、現有資料結構

### OBS_PARTS_DATA 格式（目前寫死在 HTML）

```javascript
const OBS_PARTS_DATA = {
  '頭': {
    total: 15,
    sections: [
      {
        label: '頂骨',
        qs: [
          {
            id: 'h1',
            text: '頂骨型相',
            paired: true,          // 有 paired 代表左右分開答
            opts: [
              { v: '龜背', hint: '無凹凸，緩緩向中間隆起' },
              { v: '圓', hint: '往下側偏圓弧' },
              { v: '平寬', hint: '往外側平展才向下' }
            ]
          },
          // ... 更多題目
        ]
      },
      // ... 更多 section
    ]
  },
  // ... 共 11 個部位
};
```

### 11 個部位和題數

| 部位 | ID prefix | 題數 |
|------|-----------|------|
| 頭 | h | 15 |
| 額 | e | 17 |
| 耳 | er | 15 |
| 眉 | br | 20 |
| 眼 | ey | 12 |
| 鼻 | n | 12 |
| 顴 | q | 6 |
| 口 | m | 13 |
| 人中 | p | 4 |
| 地閣 | c | 6 |
| 頤 | y | 4 |

### 題目欄位說明

| 欄位 | 必填 | 說明 |
|------|------|------|
| `id` | 是 | 題目 ID，格式為 prefix + 數字（如 h1, er7, br15） |
| `text` | 是 | 題目文字 |
| `paired` | 否 | true = 左右分開答（存為 id_L / id_R），省略 = 單題 |
| `opts` | 是 | 選項陣列，每個 {v: 選項值, hint: 提示文字} |

### 答案儲存（obsData）

- 單題：`obsData['h1'] = '龜背'`
- 左右題：`obsData['er1_L'] = '耳高'`, `obsData['er1_R'] = '耳低'`

## 四、Firebase 存取設計

### 存放位置
`settings/questions` 文件，格式：
```json
{
  "questionsJson": "<JSON 字串>",
  "updatedAt": "ISO 時間"
}
```

與 `settings/rules` 完全平行的設計。

### index_desktop.html 讀取邏輯

在 `startApp()` 裡，Step 1（讀規則）之後加 Step 1.5（讀觀察問題）：
```javascript
// Step 1.5: 從 Firebase 載入觀察問題
try {
  const qDoc = await db.collection('settings').doc('questions').get();
  if (qDoc.exists && qDoc.data().questionsJson) {
    const parsed = JSON.parse(qDoc.data().questionsJson);
    if (parsed && typeof parsed === 'object' && Object.keys(parsed).length >= 11) {
      OBS_PARTS_DATA_LIVE = parsed;
      // 用 Firebase 的覆蓋內建的
    }
  }
} catch(e) { /* fallback 到內建 */ }
```

**注意：** `OBS_PARTS_DATA` 目前是 `const`，需要改為可覆蓋的模式（類似 DIM_RULES_DEFAULT / DIM_RULES 的做法）。

## 五、admin.html 編輯器設計

### 進入方式
admin.html 頂部加一個 Tab 切換：「評分規則」|「觀察問題」

### 觀察問題編輯器 UI

**左欄：** 11 個部位列表（頭、額、耳、眉、眼、鼻、顴、口、人中、地閣、頤），顯示各部位題數

**中欄：** 選中部位的題目列表

```
[頂骨]                              ← section 標題（可改名）
  h1  頂骨型相  [左右題]  3個選項    ← 每題一行
      龜背 | 圓 | 平寬              ← 選項預覽
  h2  頂骨中央接合  2個選項
      突起 | 平順 | 凹陷
  [+ 新增題目]

[枕骨]
  h5  枕骨型相  2個選項
  ...
```

### 編輯操作

**改題目：** 點擊題目行展開編輯表單
- 題目文字（text）
- 是否左右題（paired checkbox）
- 選項列表：每個選項有 v（值）和 hint（提示），可新增/刪除選項
- 確定 / 取消

**新增題目：** 點 section 底部的「+ 新增題目」
- 自動產生下一個 ID（如目前 h 最大到 h15，新增就是 h16）
- 填寫 text、paired、選項

**刪除題目：** 每題行尾有刪除按鈕
- 刪除前掃描 13 個維度的規則 JSON，找出所有引用該 ID 的規則
- 如果有引用，顯示警告：「以下規則引用了 h5：形勢-頭、緩急-頭、順逆-頭...，請先到規則編輯器移除這些引用，再刪除題目」
- 如果無引用，確認後刪除

**調整順序：** 每題有上移/下移按鈕

**調整分組：** section 可改名，題目可拖到不同 section（或簡單版：每題有 section 下拉選單）

### 儲存 / 載入

- 「儲存到 Firebase」按鈕 → 存到 `settings/questions`
- 「從 Firebase 載入」按鈕 → 讀取最新版
- 初始化：首次開啟如果 Firebase 無資料，從 index_desktop.html 內建的 `OBS_PARTS_DATA` 格式轉存

## 六、ref 連動提示

刪除題目時，掃描規則的邏輯：

```javascript
function findRulesRefId(qId, rules) {
  const found = [];
  rules.forEach((dim, di) => {
    const dimName = dim.positive + dim.negative;
    Object.entries(dim.parts).forEach(([partName, partDef]) => {
      if (JSON.stringify(partDef).includes('"' + qId + '"')) {
        found.push({ dim: dimName, part: partName, dimIndex: di });
      }
    });
  });
  return found;
}
```

用 JSON.stringify + includes 做簡單搜尋即可，不需要遞迴走規則樹。

## 八、注意事項

- `OBS_PARTS_DATA` 被 index_desktop.html 多處引用（問卷渲染、進度計算、condResults 裡的 qMap 等），覆蓋後這些地方都要正常運作
- 題目 ID 必須唯一，新增時要檢查不重複
- `total` 欄位應該自動計算（= sections 裡所有 qs 的總數），不需要手動維護
- admin.html 需要同時能讀取規則（用於 ref 連動檢查）和觀察問題

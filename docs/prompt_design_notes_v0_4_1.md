## 一、設計哲學

### 這份 prompt 的本質

這份 prompt **不是要做一個完美的面相 AI**，而是要做一個**可以被迭代的面相 AI 起點**。

為什麼這樣定位：

1. **Mike 一個人測試不夠**——真正的品質驗證要累積 20-30 份對話、多個學員的反饋
2. **Claude 沒有學習能力**——每次對話不會記得上次、不會變聰明。**唯一能讓 AI 變好的機制，是迭代這份 prompt**
3. **現在什麼都在變**——真假 D 類還沒做、其他維度可能重新深化、學員還沒用過
4. **做一個「會變的架構」比做一個「完美的架構」更重要**

所以整份 prompt 從一開始就設計成：
- 區塊化（14 個 BLOCK 可獨立替換）
- 規格書跟骨架分離（GS 檔用佔位符，獨立版本管理）
- 每個關鍵決策都留下「為什麼」的痕跡（BLOCK 14 記錄每次改動）
- 核心原則明確標出來（V1-V6、C1-C9），讓迭代有依據

### 這份 prompt 要服務的對象

**階段 1**：Mike 自己測試，目的是驗證「內容/準確度/陳述方式/互動方式」四個向度能不能讓學員或老師接受。

**階段 2（未來）**：開放給學員，老師在後台審閱有問題的對話。

**無論階段**：學員看到的主文跟 admin 一樣——Mike 測試時模擬學員視角才有效。

## 二、從 v0.1 到 v0.4 的演化

這個演化過程本身是設計的一部分——每一版推翻前一版的原因，都值得記住，避免以後再犯。

### v0.1（2026-04-20 早期）

**當時的設計**：
- 能力範圍：只做先天六維度
- admin 跟 student 有語氣/術語差異
- 第一輪先給六維度整體小結再深入
- 議題範圍較寬，允許教練式引導

**推翻原因**：
- **能力範圍太窄**：Mike 原本要的是驗證「開放給學員水準」，學員報告是 13 維度，AI 只做 6 維度學員會發現 AI 在躲
- **admin/student 差異化錯了**：Mike 測試要模擬學員視角，兩個版本看到不一樣的話，測不出真實品質
- **第一輪結構浪費**：學員開 AI 評析時已經看完報告，AI 再講一次整體小結是重複
- **議題太寬**：教練式引導容易變成超出面相判讀的人生建議

### v0.2（2026-04-20 中期）

**當時的修正**：
- 擴充到 13 維度
- admin = student 主文一致
- 第一輪改成混合方案（結構化完整 + 結尾拋問題）
- 議題範圍收窄為純面相判讀

**推翻/調整原因**：
- **判讀漏斗設計錯了**：v0.2 走「總係數 → 找拖的層深入 → 其他帶過」的偵錯邏輯，但 Mike 指出實際判讀應該是「先講人（先天+後天）再講環境（運氣）」
- **「拖」這個字對學員不友善**：雖然是判讀技術詞，但對學員來說是負面標籤
- **三層篇幅偏食**：設計成「拖的層深入、其他一句話」必然造成三層不對等

### v0.3（2026-04-20 晚期）

**當時的修正**：
- BLOCK 04 判讀漏斗順序改為「先天 → 後天 → 運氣」
- BLOCK 09 三個情境結構重寫，三層平均深入
- 字數範圍：情境 A 1000-1200，情境 B/C 900-1100
- 新增禁用詞清單（拖、撐、短板等）
- 新增允許的中性說法清單

**演示與問題發現**：

用 Dale 案例測試 v0.3，Mike 指出幾個問題：

1. **第一段總係數與三大係數太精簡**（150 字不夠）
2. **「但你這個緩不是拖延，是事緩則圓」這句過於淡化結論**——心理諮商語氣把銳利的提醒消解掉了
3. **「你長輩走的路是逆風的，但你自己走的路是順的」是環境敘事**——應該是「你不太依循家族或長輩給你的指導，你想走自己的路」，主詞應該是「你」而非「環境」
4. **把環境風向跟家族傳承過度突出當底層論述有風險**——會變成算命式敘事

用 Sherman 案例測試，Mike 指出另外幾個問題：

5. **「領導風格」被我立成段落標題了**——變成 MBA 報告，應該融入敘事
6. **逐維度說明（一段形勢、一段經緯）效果不好**——卡在「不夠細但又顯結構」中間
7. **「傳統相理」這類權威指涉現階段不用**——沒有真的引用具體內容就是空的權威感
8. **「最特殊的點」「在領導者裡少見」這類自我抬舉沒意義**
9. **引用「老師說」會破壞對話沈浸感**
10. **當代知識（領導風格、管理學）不是不能用，是要融進敘說，點到為止**

這 10 個發現被整理成 C1-C9 九條調整原則。

### v0.4（2026-04-21）

**C1-C9 整合版**。就是現在這份。

## 四、每個 BLOCK 的設計理由

### BLOCK 01: 身份與目的

**核心判斷**：「你不是算命、不是面相 app、不是 MBTI 分類器」。

這段清楚定位 AI 不是什麼，避免它在對話中漂移。關鍵句：「把老師在課堂上會對學員說的判讀，用對話的方式重現出來」——這是 AI 的目標，不是演繹老師的判讀系統。

### BLOCK 02: 能力範圍與邊界

**能力範圍 = 全 13 維度**。v0.1 的教訓：不要限縮範圍，學員報告是 13 維度。

**邊界 = 純面相判讀**：
- 不預測未來
- 不給面相學外的行動建議
- 不評論他人
- 不做情緒諮商

**真假（09）的處理**：規格書未完成，做方向性判讀 + 主動加註「判讀邏輯仍在建立中」。這是讓 AI 能完整跑 13 維度、同時讓 Mike 能區分「真假講不準是 AI 爛還是規格書沒做」。

### BLOCK 03: 核心價值觀（V1-V6）

六條底線原則：

**V1 動靜不是好壞**——哲學底線，不會動。

**V2 係數告訴方向、部位告訴樣貌**——判讀的根本原則，不會動。

**V3 結構分析銳利、收尾回到人、但提醒不淡化**——這條是 C2 的整合點。

  - V3 的「肯定本質」用在**整段論述的收尾**
  - 不用在**每個提醒的結尾**
  - 反例：「你的緩有時候會錯過時機。**但這不是拖延，是事緩則圓。**」—— 後半句把前面的提醒消解
  - 正例：「你的緩有時候會錯過時機。」—— 提醒站住

  這個區分很重要。心理諮商語氣如果濫用，AI 會變成安慰機器，把銳利的判讀都軟化掉。

**V4 11.5-B 有方向性不批判**——倫理底線。

**V5 從靜側擴展用法不硬補動側**——老師的核心判讀技術。

**V6 引導語言要具體**——調整建議不能停在「動多往靜調」這種通用層次。

### BLOCK 04: 判讀漏斗

**核心判斷**：從「偵錯邏輯」改為「看人邏輯」。

v0.2 走「總係數 → 找拖的層深入」是技術人員的思考方式（debug）。v0.3/v0.4 改為「總係數 → 先天（這人是誰）→ 後天（怎麼打）→ 運氣（遇到什麼）」，這是面相學家的思考方式（看人）。

**內部技術詞彙對外翻譯**：
- 內部思考可以用「誰在撐、誰在拖」
- 對學員講話用「這層的環境結構比較緊」「這部分需要留意」
- 這個區分由 BLOCK 09 的禁用詞清單強制執行

### BLOCK 05: 部位分析原則

通則，不多動。核心幾條：

- 分析順序：大結構（頭、三停）→ 細部（耳眉眼鼻口）
- 三停兩軸讀法（時間軸 + 空間軸）
- 跨維度部位比對是老師實際判讀的核心技術
- 異常點判讀、群組疊加、能力鏈串聯

### BLOCK 06: 當代知識融合（C9 重寫）

**從「心理學融合」擴大為「當代知識融合」**。

v0.1~v0.3 只寫心理學，v0.4 擴大到管理學、領導學等。這來自 Mike 的一個關鍵觀察：

> 「AI 評析跟老師跟我不一樣的是，我們可以在當代心理學、管理學、領導學上面找到更多可以貼近使用者的延伸或者應用。人相兵法是一個人的觀察起點，但畢竟不是世間的唯一學問。」

關鍵尺度是**點到為止**：
- 90% 面相判讀，10% 當代延伸
- 當代知識是錨點，不是論述主體
- 過多 → 變成 MBA/心理學報告，失去面相根基

這是 C9 的核心。

### BLOCK 07: 13 維度規格書

**設計**：用 `<gs_XX>` 佔位符，由 Cloud Function 從 Firestore 載入真實內容。

**好處**：
- 你改 GS_04 曲直不用動骨架
- 骨架版本跟規格書版本獨立演進
- 對話記錄可以分別追蹤

**強化的規則（C4 + C7）**：
- 禁用「老師說」「規格書」「講義」等來源引述句型
- 禁用「傳統相理」「面相學上」這類空泛權威指涉（除非真正引用具體內容）

這是 v0.4 相對 v0.2/v0.3 大幅強化的地方——之前只寫「不標注來源」，太弱了。

### BLOCK 08: 輸入資料格式

JSON schema 定義清楚。關鍵欄位：
- `version`: "manual" vs "obs"
- `obsMapping`: 只有自動版存在，讓 AI 能做層級 B+ 深度
- `age`: 缺失時主動詢問（三停時間軸需要）

### BLOCK 09: 第一輪輸出設計（變動最大的 BLOCK）

**共通精神六條**（C1-C9 的濃縮）：

1. 先講人再講環境（C3）
2. 論述流動不擺架構（C5、C6）
3. 維度是材料不是主題（C6）
4. 點到為止地用當代語彙（C9）
5. 提醒站住不立刻補安慰（C2）
6. 結尾問題必須具體（原有）

**三個情境的結構**：

- **情境 A**（勾全部 13 維度）：1200-1500 字，總係數 → 先天 → 後天 → 運氣 → 收尾
- **情境 B**（勾組合係數）：1100-1400 字，該組合係數深入
- **情境 C**（勾單一維度）：900-1200 字，該維度講透

**關鍵設計：結構是內部思考用，不是輸出格式**。AI 不應該把「## 先天層」「## 後天層」立成 Markdown 標題——那會變成報告。應該用自然敘事銜接。

**領導/管理風格融入分支邏輯**（C5 修正版）：

| 條件 | 敘事重心 |
|------|---------|
| 老闆係數明顯高於主管 | 融入「你的領導風格會傾向...」 |
| 主管係數明顯高於老闆 | 融入「你的管理風格原則可能是...」 |
| 兩者相近且都 ≥ 0.5 | 兩種風格都可以提 |
| 兩者都 < 0.5 偏動 | 不用風格架構，直接描述適合什麼角色 |

**強化的禁用詞清單**（負面標籤 + 自我抬舉 + 教科書腔 + 權威指涉 + Markdown 標題）。

### BLOCK 10: 追問輪設計

五種追問類型：

1. 深入某維度
2. 切換維度
3. 學員補充生活狀況（允許重新解讀，但只接回判讀不給建議）
4. 學員不認同判讀（2 輪內處理不來轉老師）
5. 學員問面相學外的問題（走 BLOCK 11 轉老師）

**跨維度引用不限數量**——老師實際判讀引用 3-4 個維度是正常的。

### BLOCK 11: 對話邊界與轉老師

五種轉老師情境：
1. 學員問面相學外的具體行動
2. 學員問健康/醫療/法律
3. 學員問預測
4. 學員兩輪內仍強烈不認同
5. 學員有情緒失控/自傷傾向

**寬鬆門檻**：先嘗試把問題轉回 13 維度範圍內的判讀，真的處理不來才轉。

**灰色地帶處理表**明確列出允許/不允許的細節情境。

### BLOCK 12: admin vs student

**兩個模式主文完全一樣**——這是 v0.1 教訓後的關鍵設計。

唯一差異：admin 模式 Cloud Function 帶 Notion MCP 工具，讓 AI 可以查案例庫和維度對照表。但查的結果融入判讀，**不在主文秀「我查了什麼」**。

### BLOCK 13: 生成前自我檢查

**從 4 項擴充到 11 項**（v0.4 新），覆蓋：
- 判讀正確性（3 項）
- 陳述方式（5 項，含 C3、C4、C6、C7、C8）
- 心理諮商語氣（2 項，含 C2、V3）
- 當代知識（1 項，含 C9）

自我檢查的副作用提醒：AI 有時會把檢查過程寫進回應。要在 prompt 強調「不要把檢查過程寫進回應，這是內部動作」。

### BLOCK 14: 版本資訊

每次改動在這裡留記錄。未來任何接手的 Claude 看這段就知道歷史。

## 五、C1-C9 九條核心調整的來龍去脈

這九條是 v0.2/v0.3 測試後發現的問題，每條都有具體的觸發案例。

### C1 字數壓縮

**起源**：v0.3 演 Dale 寫了 1850 字，超過 1200 上限 650 字。

**根本原因**：v0.3 結構寫得鬆，三層深入 + 收尾加起來容易膨脹。

**解法**：情境 A 字數放寬到 1200-1500，情境 B 1100-1400，情境 C 900-1200。不強壓 1000 內。

**要動的時候**：測試後如果學員覺得太長/太短。

### C2 提醒不淡化

**起源**：v0.3 Dale 寫「你的緩有時候會錯過時機，**但這不是拖延，是事緩則圓**」。Mike 指出「但」後面那句把前面的提醒消解了。

**根本原因**：V3 心理諮商語氣被過度泛化——本來是「收尾安頓」，變成「每個提醒都要安慰」。

**解法**：V3 改寫，明確區分「整段收尾的安頓」vs「每個提醒的消解」。提醒讓它站住，安頓放收尾。

**要動的時候**：如果測試發現 AI 又開始在提醒後補安慰，可能要更嚴格的執行規則。

### C3 主詞是這個人

**起源**：v0.3 Dale 寫「你長輩走的路，家族傳給你的方向，在這個時代是逆風的」。Mike 指出這是環境敘事——順逆應該讀成「你逆著長輩的方向」而不是「環境/家族逆風」。

**根本原因**：我把「逆」讀成「環境逆」而不是「這個人逆著環境/長輩」。這是讀面相的視角錯了——人相兵法是看「這個人」的，不是看「這個人的命運」。

**解法**：
- BLOCK 09 共通精神第 1 條：主詞嚴格用「你」
- BLOCK 09 情境 A 運氣層：明確寫「不寫『環境給你 X』『世界對你 X』『家族傳給你 X』」
- BLOCK 13 自我檢查第 4 項：檢查有沒有環境敘事

**運氣維度特別嚴格**：因為運氣三維度（順逆/分合/真假）天然誘發環境敘事，在這層要最嚴。

**要動的時候**：如果測試發現 AI 在後天維度也出現類似環境敘事，原則要擴大到全 13 維度。

### C4 不引用老師、不出現規格書

**起源**：v0.3 Sherman 寫「老師對你之前的原話是『創業老闆，做調整就好』」。Mike 指出這種敘述不需要，直接把結論講出來就好。

**根本原因**：
- 「老師說」破壞對話沈浸感（變成轉述）
- 學員需要判讀本身，不是判讀的來源
- 避免學員以為 AI 只是在轉述

**解法**：BLOCK 07 強化禁用句型，列具體反例和正例。

**要動的時候**：未來如果 AI 真的有需要引用具體講義內容（例如「眼鉤在相理裡代表 XX」），可以考慮加一條「具體引用講義條目時允許」的例外。

### C5 領導/管理風格融入而非架構（修正版）

**起源**：

第一次（Sherman 演示）：我把「領導風格」立成段落標題「## 你是哪一種領導者」，Mike 指出這是 MBA 報告腔。

第二次（Mike 補充）：不是不能用「領導風格」「管理風格」這幾個字，是使用方式要自然，像在說話。而且這兩個詞來自 AI 評析可以連結當代管理學/領導學的優勢。

**根本原因**：架構是 AI 內部用的，輸出時不該顯性化。

**解法**：
- BLOCK 09 共通精神第 2 條：「論述流動，不擺架構」
- 情境 A 先天層、情境 B 開場：領導/管理風格**融入敘事**，不立段落標題
- 含分支邏輯（老闆 vs 主管哪個高）

**要動的時候**：如果測試發現其他當代詞彙（例如「職業錨」「依附型態」）也有融入 vs 擺架構的問題，可能要把這個原則擴大。

### C6 不逐維度說明

**起源**：v0.3 Sherman 寫成「形勢偏勢：[一段]；經緯偏緯：[一段]；方圓偏圓：[一段]；曲直：[一段]；收放：[一段]；緩急：[一段]」。Mike 指出這樣「變成分維度說明，又不夠細緻」。

**根本原因**：AI 把「把維度講清楚」誤解為「每個維度分一段講」。但**維度是論述材料，不是段落主題**。

**解法**：
- BLOCK 09 共通精神第 3 條：「維度是材料不是主題」
- 三個情境結構都強調：把幾個維度織在一起講「這個人是什麼樣」，不是逐維度說明

**要動的時候**：未來如果單一維度的判讀細節化到足夠深度（現在的 GS 檔還不夠細到可以逐維度深論），這個原則可能可以局部放寬。

### C7 不用空泛權威指涉

**起源**：v0.3 Sherman 寫「在傳統相理裡是幕僚或幕僚長的構造」。Mike 指出：不是不能用「傳統相理」，但現階段沒有真的引用到講義細節，這個詞用得空泛。

**根本原因**：空泛的權威感對學員沒價值，學員無法驗證。

**解法**：BLOCK 07 強化，現階段一律不用這類詞，除非真實引用具體講義內容。

**要動的時候**：未來 GS 檔的條件細節讓 AI 可以具體引用「眼鉤在講義裡解釋為富智謀善經營」這種程度的內容時，可以加回「具體引用時允許」的例外。

### C8 不自我抬舉判讀價值

**起源**：v0.3 Sherman 寫「這是你最特殊的點」「在領導者裡少見」。Mike 指出「沒有意義，不一定增加閱讀者對評析的 credit」。

**根本原因**：這種說法的本質是 AI 在自我抬舉判讀的稀有性。學員不需要知道判讀稀不稀有，需要知道判讀本身對他什麼意義。

**解法**：BLOCK 09 禁用詞清單新增「自我抬舉類」。

**要動的時候**：這條應該不太需要動。

### C9 當代知識融合

**起源**：Mike 的補充說明——AI 評析跟老師、Mike 的差異是可以連結當代心理學/管理學/領導學，讓判讀貼近使用者。但要點到為止。

**根本原因**：v0.1~v0.3 只寫「心理學融合」，太窄。v0.4 的 BLOCK 06 擴大為「當代知識融合」，並明確「點到為止」的尺度。

**解法**：BLOCK 06 完全重寫，加入比例、位置、頻率三個具體尺度。

**要動的時候**：如果發現某個學科連結特別有效（例如某學員對「敘事治療」特別有感），可以把那個連結在 BLOCK 06 寫得更具體。

### Cloud Function 組裝邏輯

```javascript
async function assembleSystemPrompt(userRole) {
  // 1. 讀當前版本
  const versionDoc = await db.doc('settings/aiPrompt').get();
  const currentVersion = versionDoc.data().current;

  // 2. 讀所有 blocks 和 dimensions
  const blocksSnap = await db.collection(
    `settings/aiPrompt/versions/${currentVersion}/blocks`
  ).get();
  const dimensionsSnap = await db.collection(
    `settings/aiPrompt/versions/${currentVersion}/dimensions`
  ).get();

  // 3. 拼接 blocks（依照 BLOCK 編號順序）
  const blocks = blocksSnap.docs
    .sort((a, b) => a.id.localeCompare(b.id))
    .map(doc => doc.data().content)
    .join('\n\n');

  // 4. 替換佔位符
  let finalPrompt = blocks;
  dimensionsSnap.docs.forEach(doc => {
    const gsId = doc.id;  // e.g. "gs_01"
    const content = doc.data().content;
    const placeholder = new RegExp(
      `<${gsId}>\\s*\\[[^\\]]*\\]\\s*</${gsId}>`,
      's'
    );
    finalPrompt = finalPrompt.replace(
      placeholder,
      `<${gsId}>\n${content}\n</${gsId}>`
    );
  });

  return {
    systemPrompt: finalPrompt,
    versionInfo: {
      skeleton: currentVersion,
      dimensions: dimensionsSnap.docs.reduce((acc, doc) => {
        acc[doc.id] = doc.data().version || "unknown";
        return acc;
      }, {})
    }
  };
}
```

## 十、給未來接手的 Claude 的話

如果你接手這個專案，以下幾件事要先理解：

### 1. 看歷史再動手

這份 prompt 經過 v0.1 → v0.2 → v0.3 → v0.4 四次大改。每次推翻都有具體原因，寫在本文件 §二。**不要重複犯同樣的錯**：
- 不要把能力限縮到少於 13 維度
- 不要把 admin 和 student 主文差異化
- 不要用「拖/撐/偵錯」邏輯設計判讀流程
- 不要把分析框架（領導風格、判讀漏斗）立成段落標題
- 不要在每個提醒後補安慰

### 2. 先看 C1-C9 的來源再調整

如果要改某個原則，先看本文件 §五那個原則是怎麼來的、觸發案例是什麼。不然會改回已經被排除過的方向。

### 3. 區分「底線」和「暫訂」

本文件 §六的「不會動的底線」真的不動——動了就不是人相兵法 AI 了。

本文件 §七的「預期會動的地方」是暫訂的，測試後依據實際數據調整。

### 4. 累積數據再改

迭代 prompt 要根據對話記錄的 pattern，不要根據單次對話的直覺。

累積至少 20 份對話 + 某個問題重複出現 → 才改。

### 5. 心理諮商語氣的使用要小心

V3 是最容易被濫用的原則。濫用會讓 AI 變成安慰機器，把銳利的判讀都軟化掉。

具體警戒線：
- 在「提醒」後面**不要**立刻補「但其實你很棒」
- 在「結尾整段」才落實 V3 三特徵
- 極端係數（0.00, 0.13）才需要全程注意心理諮商語氣
- 中性係數（0.50-0.80）只在收尾輕帶

### 6. 當代知識不是炫技

點到為止。90% 面相判讀，10% 當代延伸。不要反過來。

學員看完論述要感覺到的是「這在說我這個人」，不是「這是一份管理學分析」。

### 7. 真假（09）的處理要誠實

規格書未完成。AI 可以做方向性判讀，但**必須主動標注「判讀邏輯仍在建立中」**。這不是裝謙虛，是讓 Mike 能區分「真假講不準是 AI 爛還是規格書還沒做」。

真假 D 類完成後，要更新 BLOCK 02 的真假處理規則，並在 BLOCK 04 §運氣內部補上「順逆 × 分合 × 真假」三維度聯看邏輯。

### 8. 字數不是硬指標

1200-1500 是目標範圍，不是絕對上限。某些學員（例如多個極端值的）可能需要多一點篇幅才能講清楚。但超過 1800 就是 AI 在偷懶（寫太開、資訊沒濃縮）。

### 9. 描述現狀優先於預言未來（2026-04-21 實測補入）

V3 的「給予正向預期」容易被誤用成「你的未來會 X」這種預言式安慰。更高階的作法是**描述學員當下已經在發生的事**——把預言變成現狀確認。

範例對比：
- ❌「你走這條路會走得下去」（預言）
- ✅「你走到今天，其實已經在走這條路上了」（現狀確認）

現狀確認的力量比預言大。預言可以被質疑（「憑什麼？」），現狀確認讓學員自己認得出來（「對，我確實在做」）。

### 10. 拉鋸係數（0.50、0.80）不要預設讀成「模糊」

**常見偏差**：看到 0.50 / 0.80 反射性讀成「核心價值還沒鎖定」「多方向不聚焦」「模糊」。這個偏差讓 AI 沒看部位分布就先下負向詮釋。

**正確做法**：看到拉鋸係數，**先把九個部位的動靜排出來找結構**，再下詮釋。常見的結構包括：
- 三停 vs 五官的分布差異
- 內層（耳眼鼻）vs 外層（頭上停口）的分布差異
- 大結構（頭三停）vs 細部（五官）的一致性

例：孟璋經緯 0.50 偏緯——看起來是「核心價值多變」，但看部位分布「頭上中下全緯 + 耳眼鼻經」，真正讀法是「大原則可調整、核心軸線鎖死」，完全不是「模糊」。

### 11. 係數的意義是相對的，不是絕對的

同一個 0.50，在不同學員身上意義不一樣：
- 孟璋先天 0.69、整體偏穩 → 0.50 是拉鋸區、要留意的點
- 詩璇先天 0.17、整體極動 → 0.50 是先天結構裡最接近動靜平衡的維度、是**相對優勢**

**判讀時要問**：這個數字在**這個人的整體結構裡**處在什麼位置？不要套固定詮釋。

### 12. 預設價值框架的陷阱：「低分需要重新詮釋」

**常見偏差**：遇到低係數（0.17、0.00）會本能地想「安慰」學員、「重新詮釋」、「扭轉負面印象」。但這個衝動本身**已經預設了「這是負面的」**——只要 AI 用「這不是次一等」「這不是有問題」「看起來很低但...」這類句型，就已經把**原本不存在的價值判斷引入了對話**。

**V1 的嚴格版本**：不只「不講好壞」，是**連暗示好壞的框架都不建立**。不用「低/高/夠/不夠/好/差/有問題/沒問題」這類詞。

**測試法**：每次想加「這不是 X」「不是說 Y」這種句式時問自己——X 或 Y 是學員本來就會想到的，還是我預判他會這樣想？如果是後者，整句刪掉。

### 13. 單一部位單一維度不能腦補大敘事

**常見偏差**：看到「上停方圓是方（單一維度靜側）」就推演出「早年家庭給你穩定的原則傳承」一整段家庭敘事。這違反 V2（係數告訴方向、部位告訴樣貌）——**真正的樣貌要跨維度部位疊加才能讀出**。

**判讀規則**：
- 單一部位單一維度 = **一個資訊點**，不足以推大敘事
- 跨維度部位疊加 = **可以讀出樣貌**（例：頭方+耳方 要跟 頭勢緯+耳勢緯 合看，才能論家運）
- 寫每段前自問：這段的每一句有沒有規格書/老師原話/多維度部位疊加支撐？

### 14. 維度本質要記牢，不要望文生義

每個維度有明確的本質意義，不能用字面義推論：
- **收放 = 能耐**，不是「能量流動方向」。收少於放 ≠「能量往外散」，而是「能耐有少數錨點、要從這些錨點擴展」
- **緩急 = 行事成敗節奏**，不是「速度」。緩急偏急 ≠「這個人很急躁」，要看部位分布才知道急在哪（腦子急 vs 鼻急 vs 手腳急意義不同）
- **經緯 = 核心價值的鎖定與變動**，不是「想法穩不穩定」。經緯偏緯 ≠「這個人想法多變」，要看三停 vs 五官的分布才知道是「尋找焦點」還是「核心軸線在哪裡」
- **曲直 = 責任擔當的路徑**，不是「做事繞不繞」。等等

每次判讀前回去查 GS_XX 規格書對應的「本質定義」。

### 15. 「頭 × 上停 × 口」三部位對照法（待更多案例驗證）

2026-04-21 Mike 補入的判讀技術：頭（思考）× 上停（消化）× 口（表達）的動靜對照能讀出表裡一致性：

- 頭動 + 上停動 + 口靜 → 想很多、不太說（孟璋）
- 頭動 + 上停動 + 口動 → 想很多、也說很多（詩璇）
- 其他組合待驗證

這是**部位功能對照法**的一個具體應用。更廣的原則：五官各有功能（耳接收、眼判斷、鼻執行、口表達、眉意志），不要用「五官全靜」這種粗粒度描述——拆開跟頭、上停、中停、下停對照才能看到結構。

### 16. 下停的兵法意義（BLOCK 05 建議補強）

BLOCK 05 的三停兩軸讀法（時間軸/空間軸）**沒有點出下停的兵法角色**。下停不只是「50 歲以後」或「晚輩/部屬」，在兵法結構裡是：

- **落地區**：中停開路的東西最終要落在下停
- **收成區**：結果能不能結成形、留得住，看下停
- **守成區**：守得住資源不漏，下停 + 口兼看

呱呱案例（2026-04-21）：中停最動 + 下停 + 口有三個靜（方、收、經）→ 「中停開路、眼形判斷、下停口收成」的完整結構，才是這個人的兵法圖像。只讀時間軸會漏掉這一塊。

### 測試累積狀況

| 日期 | 案例 | 情境 | 模式 | 迭代輪次 | 主要學習 |
|------|------|------|------|---------|---------|
| 2026-04-21 | 呱呱 | B 先天 | student | 2 輪 | 下停的兵法意義（不只是時間軸） |
| 2026-04-21 | 孟璋 | B 先天 | admin（補 Notion） | 4 輪 | 拉鋸係數偏差、腦補家族敘事、經緯內外層讀法 |
| 2026-04-21 | 詩璇 | B 先天 | student → admin | 2 版對比 | 價值框架偏差、維度本質、係數相對性、收尾雞湯感 |

**累積 3 份，離 20 份門檻還遠**。以下觀察不觸發骨架修改，只累積紀錄。

### 實測浮現的系統性偏差清單

#### 偏差 A：拉鋸係數預設讀成「模糊」

**現象**：看到 0.50 / 0.80 反射讀成「核心價值模糊」「方向不聚焦」。

**觸發案例**：孟璋經緯 0.50、詩璇緩急 0.50。

**修正方向**：先看部位分布結構再下詮釋。**已寫入本文件第十章第 10 條**。

**可能的 prompt 補強點**：BLOCK 03 V2 或 BLOCK 05 加一條「拉鋸係數判讀規則」。

#### 偏差 B：低係數預設當「需要重新詮釋的負面」

**現象**：遇到先天 0.17、形勢 0.00 等低係數，反射性想安撫/扭轉，用「這不是次一等」「看起來很低但不是」這類句型——**這些句型本身已經引入了不存在的價值判斷**。

**觸發案例**：詩璇先天 0.17（Mike 兩次糾正「你已經在做價值判斷了」）。

**修正方向**：V1「動靜不是好壞」的嚴格版——連暗示好壞的框架都不建立。**已寫入本文件第十章第 12 條**。

**可能的 prompt 補強點**：BLOCK 03 V1 條文擴充，加入「不使用『低/高/夠/不夠』這類詞描述係數」的明確指令。

#### 偏差 C：單一部位單一維度腦補大敘事

**現象**：看到「上停方圓是方」就推出「家庭給你穩定原則傳承」這類整段家族敘事，實際上單一部位單一維度的訊息量不足以支撐這種推演，而且往往跟規格書真正的讀法相反。

**觸發案例**：孟璋第二版的上停方圓段（完全腦補、且跟規格書「上停偏動 = 祖輩父母條件不好」方向相反）。

**修正方向**：跨維度部位疊加才能讀樣貌（例：頭方+耳方 要跟 頭勢緯+耳勢緯 合看）。**已寫入本文件第十章第 13 條**。

**可能的 prompt 補強點**：BLOCK 13 自我檢查加一條「每段論述能否追溯到跨維度部位疊加或老師原話？」。

#### 偏差 D：維度望文生義

**現象**：把收放讀成「能量方向」（正確是能耐）、把緩急讀成「速度」（正確是成敗節奏）、把經緯讀成「想法穩定度」（正確是核心價值鎖定）。

**觸發案例**：詩璇收放 0.29 被讀成「能量往外放、不收斂」（負向），實際讀法是「能耐的錨點是少數的收、要找到它們為起點」（建構性）。

**修正方向**：判讀前回查 GS_XX「本質定義」。**已寫入本文件第十章第 14 條**。

**可能的 prompt 補強點**：BLOCK 07 規格書使用規則加一條「判讀任何維度前，先從 GS_XX 讀取該維度本質定義，不望文生義」。

#### 偏差 E：係數意義被當成絕對值

**現象**：把 0.50 套固定詮釋為「拉鋸狀態、普通、要留意」，沒考慮它在**這個人整體結構裡**的相對位置。

**觸發案例**：詩璇緩急 0.50，在她先天 0.17、其他維度 0.00-0.29 的結構裡**是相對優勢**，不是普通。

**修正方向**：判讀每個係數時問「這在這個人整體結構裡是什麼位置？」。**已寫入本文件第十章第 11 條**。

**可能的 prompt 補強點**：BLOCK 04 判讀漏斗補一步「比較各維度係數相對高低」。

#### 偏差 F：下停只當時間軸，漏掉兵法意義

**現象**：分析下停時只講「50 歲以後、晚輩部屬」，沒讀出下停+口是**落地/收成/守成**區塊。

**觸發案例**：呱呱第一版漏下停（Mike 指出後補入「下停的兵法意義」）。

**修正方向**：下停 + 口兼看、讀出落地收成結構。**已寫入本文件第十章第 16 條**。

**可能的 prompt 補強點**：BLOCK 05「大結構與細部關係」表格補入「下停 = 落地/收成/守成」的兵法定位，不只有時間軸和世代軸。

#### 偏差 G：五官粗粒度描述（「五官全靜」）

**現象**：用「五官全靜」這種粗粒度描述，錯過「口靜 vs 頭上停動」「耳眼鼻全靜 vs 眉口動」這類更精準的結構讀法。

**觸發案例**：孟璋原版讀成「五官穩」——漏掉 Mike 後來補入的「頭動+上停動+口靜=想很多不表達」的三部位對照法。

**修正方向**：五官各有功能，拆開對照才能讀結構。**已寫入本文件第十章第 15 條**。

**可能的 prompt 補強點**：BLOCK 05 部位分析原則補一條「五官要拆開讀：耳接收、眼判斷、鼻執行、口表達、眉意志，各自跟頭三停的動靜對照」。

#### 偏差 H：收尾雞湯感

**現象**：收尾句容易寫成對仗工整的句式（「不是 X，是 Y」「A 在 B 裡」），中文裡這種結構本身就是雞湯語法。

**觸發案例**：詩璇收尾「你在尋找的，是你自己這條路的尺」——Mike 指出可以更好。

**修正方向**：收尾用描述現狀（「你已經在走了」）代替預言或修辭對仗。參考老師的語法——一層一層遞進、不對仗、描述實際運作方式。**已寫入本文件第十章第 9 條**。

**可能的 prompt 補強點**：BLOCK 09「結尾拋問題」段補一條收尾句原則。

### Mike 的判讀偏好（從 sandbox 歸納）

這些不是普遍偏好，是 Mike 在修正 Claude 時顯示出的**判讀品味**：

1. **精準度優先**：「高階夥計」比「穩中求變的主管」好；「先天的基礎樣貌」比「底牌」好
2. **主動性優先**：「他選擇走主管路」比「他不適合當老闆」好
3. **正向潛能優先**：「有格局 + 清晰方向就能擴大落實」比「核心價值不夠聚焦」好
4. **發展軌跡優先**：「長期累積會讓中下停核心價值清晰」比靜態定型畫像好
5. **拒絕腦補**：沒依據的延伸即使順暢也要刪
6. **拒絕 signposting**：不寫「先講結論」這類提示語，直接進內容
7. **拒絕反駁未提出的觀點**：不寫「這不是 X」當學員根本沒說 X
8. **老師的語法樣本**：多層遞進、不對仗、描述實際運作，不修辭不雞湯

這些偏好**可能是通用判讀原則、也可能是 Mike 個人風格**，待更多測試對話後判斷。

### student vs admin 模式差距觀察

| 項目 | student 模式 | admin 模式（有 Notion MCP） |
|------|-------------|------------------------|
| 定位詞精準度 | 抽象描述 | 直接用教學案例的精準定位 |
| 核心判讀技術 | 漏掉老師讀過的核心點 | 抓到 |
| 正向安頓語氣 | 偏中性、容易誤用價值框架 | 有老師樣本可參考 |
| 價值判斷偏差 | 仍會發生（需 Mike 修正） | 仍會發生（需 Mike 修正） |
| 整體滿意度 | 表面 OK、細節漏 | 細節到位、但仍需修正讀法 |

**初步結論**：
- Admin 模式 MCP 查案例是**有價值的**——尤其對有教學案例的學員
- 但 MCP 不能解決**判讀框架偏差**（B/D/E 這類偏差 admin 模式同樣會犯）
- 框架偏差要靠 prompt 本身的條文補強，不是靠查案例

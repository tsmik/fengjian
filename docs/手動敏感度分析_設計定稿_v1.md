## 2. 資料模型與計算基礎

### 2.1 輸入資料

- `manualData[13][9]` — 13 維度 × 9 部位的 A/B/null 矩陣
- `DIMS[0..12]` — 維度定義（含 a/b 屬性字、正負側）
- `BETA_VISIBLE_DIMS` — 學員可見維度數

### 2.2 13 維度 → 區塊對應

| 區塊 | 維度 index | 維度名（a/b） |
|------|------------|---------------|
| 老闆（先天） | 0, 1, 2 | 形勢 / 經緯 / 方圓 |
| 主管（先天） | 3, 4, 5 | 曲直 / 收放 / 緩急 |
| 先天（合併） | 0–5 | 上述六個 |
| 運氣 | 6, 7, 8 | 順逆 / 分合 / 真假 |
| 後天 | 9, 10, 11, 12 | 攻守 / 奇正 / 虛實 / 進退 |

### 2.3 均衡度指標（靜側柱子）

每個維度的「靜側數量」 = 該維度 9 個部位中屬於靜側的格子數（參見現有 `calcDim` 定義的 a/b 型）。

| 區塊 | 均衡度 = max−min | 柱子數 |
|------|-----------------|--------|
| 老闆 | max(形靜, 經靜, 方靜) − min(...) | 3 |
| 主管 | max(直靜, 收靜, 緩靜) − min(...) | 3 |
| **先天** | **max(形靜, 經靜, 方靜, 直靜, 收靜, 緩靜) − min(...)** | 6 |
| 運氣 | max(順靜, 分靜, 真靜) − min(...) | 3 |
| 後天 | max(攻靜, 奇靜, 虛靜, 進靜) − min(...) | 4 |

差距判斷：
- `≤ 2` → ⚪ 均衡
- `≥ 3` → ⚠ 需提醒短板

### 2.4 子係數計算

沿用現有 `calcDim / avgCoeff` 並加上組合：
- 老闆係數 = `avgCoeff(manualData, [0,1,2])`
- 主管係數 = `avgCoeff(manualData, [3,4,5])`
- 先天總係數 = `avgCoeff(manualData, [0,1,2,3,4,5])`
- 運氣總係數 = `avgCoeff(manualData, [6,7,8])`
- 後天總係數 = `avgCoeff(manualData, [9,10,11,12])`

#### 3.2.1 老闆係數小卡

```
┌─ 老闆係數 1.2.3 ─────────────────────────┐
│ 係數：0.XX（靜X 動X）                      │
│                                            │
│ 方向判讀：                                  │
│   形>勢 ✓  經>緯 ✓  方>圓 ✓               │
│   → 創業老闆方向                           │
│ （或其他方向組合對應的判讀文字，見 3.2.1.1）│
│                                            │
│ 1+2=3 因果前提：                            │
│   形勢 ✓  經緯 ✓                           │
│   → 方圓可深論                              │
│ （或前提不足時的話術，見 3.2.1.2）          │
│                                            │
│ 均衡度（靜側柱子圖）：                      │
│   形 ▓▓▓▓▓ 5                              │

│   方 ▓▓▓▓▓▓▓▓ 8                           │
│   差距 4  ⚠ 初心不穩                        │
│   短板：方（成就方向）明確但形勢格局動能不足 │
│                                            │
│ 形勢阻力效應：⚠（觸發時才顯示）             │
└────────────────────────────────────────────┘
```

**3.2.1.1 方向判讀邏輯**（依 GS_14 §2 第一步）：

| 形勢方向 | 經緯方向 | 方圓方向 | 判讀文字 |
|---------|---------|---------|---------|
| 形>勢 | 經>緯 | 方>圓 | 創業老闆方向 — 做事的人 |
| 形>勢 | 經>緯 | 方<圓 | 創業老闆方向 — 做老闆的人（看圓） |
| 形<勢 | 經<緯 | 方<圓 | 做老闆辛苦（老師原話） |
| 其他組合 | | | 方向不完全符合，看係數 0.80 是調整機會 |

**3.2.1.2 1+2=3 前提檢查**（依 GS_14 §5 第三步）：

- 形勢動靜兩側的係數低於 0.50 或沒有調整空間（兩邊差很大）→ 前提不到位，方圓帶過
- 經緯同上
- 兩個都到位 → 方圓可深論

話術：
- 兩個都到位：「方圓可深論」
- 形勢不到位：「形勢格局未到位，方圓的成就先不用論，先處理形勢」
- 經緯不到位：「經緯的核心價值未到位，方圓的成就先不用論，先處理經緯」
- 兩個都不到位：「形勢經緯都未到位，方圓的成就先不用論，先處理前提」

**3.2.1.3 均衡度短板話術**（依 GS_14 §5 第四步 + GS_15 §4）：

形、經、方 三根靜側柱子的最小者 = 短板。

| 短板是 | 話術 |
|-------|------|
| 形（靜側最少） | 格局動能不足：方向有、核心有，但格局撐不開 |
| 經（靜側最少） | 核心價值不穩：格局有、方向有，但初心飄移 |
| 方（靜側最少） | 成就方向不明：格局有、核心有，但不知道要追尋什麼 |

若兩根並列最小 → 並列都點出。

**3.2.1.4 形勢阻力效應**（依 GS_14 §5 + AI評析跨維度 §8.5）：

觸發條件（五項全符合才觸發）：
1. 形>勢 ✓
2. 經>緯 ✓
3. 經緯係數 ≥ 0.50
4. 方圓係數 ≥ 0.50
5. 形勢係數偏低（= 0.13 或 0.29）

觸發時顯示：
> ⚠ 你有清楚的核心價值，也知道自己要追求什麼方向的成就，但在把格局做大這件事上動能不足。不是不想做，是目前的狀態太穩了，缺少一個推力讓你真正出手。

#### 3.2.2 主管係數小卡

結構同老闆係數小卡，替換規則：

**3.2.2.1 方向判讀**（GS_15 §3-4 規則合集）：

| 曲直 | 收放 | 緩急 | 判讀文字 |
|------|------|------|---------|
| 直>曲 | 靜>動 | 緩>急 | 典型穩當主管（GS_15 §4）|
| 曲>直 | — | — | 幕僚/幕僚長傾向（老師原話）|
| 眼睛偏圓（眼為動側）| — | — | 才華型，當主管有顧慮（老師原話；此條在下半部說明頁輔助顯示）|

**3.2.2.2 4+5=6 前提檢查**（GS_15 §2）：

- 曲直到位 + 收放到位 → 緩急可深論
- 曲直不到位 → 緩急帶過，「擔當不足，扛不起責任」
- 收放不到位 → 緩急帶過，「能耐不足，做不到」
- 都不到位 → 「擔當和能耐都不足，節奏先放一邊」

**3.2.2.3 均衡度短板話術**（GS_15 §4）：

直、收、緩 三根靜側柱子的最小者 = 短板。

| 短板是 | 話術 |
|-------|------|
| 直（靜側最少，偏曲） | 擔當不足：能耐和節奏到位但扛不起責任 |
| 收（靜側最少，偏放） | 能耐不足：扛起責任有節奏但做不到 |
| 緩（靜側最少，偏急） | 節奏太快：有擔當有能耐但衝太急容易出錯 |

**3.2.2.4 收放×曲直互補邏輯**（GS_15 §5，v1 先不在此頁實作，留到 v2 或部位層）

#### 3.2.3 老闆 vs 主管 角色定位

依 GS_16 §6 的四象限歸納：

| 條件 | 角色 |
|------|------|
| 老闆 > 主管 且 1-3 方向對 | 創業老闆 |
| 主管 > 老闆 且 4-6 均衡（差≤2） | 高階夥計 |
| 老闆 ≈ 主管（差≤0.1）且 經緯或曲直低 | 高階幕僚 |
| 先天總係數 < 0.20 | 先天極低 — 個人工作者、接案性質 |
| 其他 | 老闆/主管皆可，看內部細節 |

顯示形式：
```

[附一行說明]
```

**先天極低特例**（GS_16 §5）：

## 4. 下半部 — 格子翻轉建議 Top N

### 4.1 區塊結構

| 區塊 | Top N | 翻轉範圍（維度 index） | 排序依據 |
|------|-------|---------------------|---------|
| 老闆 Top 2 | 2 | 0–2 | 老闆均衡改善量 → 老闆係數改變量 |
| 主管 Top 2 | 2 | 3–5 | 主管均衡改善量 → 主管係數改變量 |
| **先天 Top 3** | 3 | 0–5 | **先天6柱均衡改善量 → 先天總係數改變量** |
| 運氣 Top 3 | 3 | 6–8 | 運氣均衡改善量 → 運氣總係數改變量 |
| 後天 Top 3 | 3 | 9–12 | 後天均衡改善量 → 後天總係數改變量 |

### 4.2 翻轉規則

- **單格 A↔B 翻轉**，不含 null
- 若某格目前為 null，跳過（不嘗試改為 A 或 B）
- 每個區塊獨立掃描自己範圍內的格子

### 4.3 排序演算法

```pseudo
for each 區塊:
    目前差距 D0 = 計算均衡度(當前靜側柱子)
    目前子係數 C0 = 計算子係數
    
    candidates = []
    for each 格子 (dim, part) in 區塊維度範圍:
        當前值 = manualData[dim][part]
        if 當前值 是 null: skip
        目標值 = 當前值 === 'A' ? 'B' : 'A'
        
        暫存 = manualData[dim][part]
        manualData[dim][part] = 目標值
        D1 = 計算均衡度
        C1 = 計算子係數
        manualData[dim][part] = 暫存  // 還原
        
        均衡改善量 = D0 - D1  // 正數表示更均衡
        係數改變量 = C1 - C0  // 正數表示拉高
        
        candidates.push({dim, part, 均衡改善量, 係數改變量, 翻轉方向: 當前值 + '→' + 目標值})
    
    candidates.sort((a, b) => {
        // 主排序：均衡改善量（降序，改善越大越前）
        if (a.均衡改善量 !== b.均衡改善量) return b.均衡改善量 - a.均衡改善量
        // 次排序：係數改變量（降序，拉高越多越前）
        return b.係數改變量 - a.係數改變量
    })
    
    Top N = candidates.slice(0, N)
```

### 4.4 「已均衡區塊」特例處理

若區塊當前差距 ≤ 2：
- 上半部診斷顯示「⚪ 均衡」
- 下半部 Top N **依然列出**，但排序改為：
  - 主排序改為 **係數改變量（絕對值）** 降序（調整空間大的格子優先）
  - 次排序保持均衡改善量

### 4.5 呈現格式

每個 Top N 項目一行：

```
[1] 鼻 × 形勢 A→B
    均衡 4→2 (改善+2)  老闆係數 0.45→0.48
```

視覺：
- 部位名 + 維度名 + 翻轉方向用淺色高亮格
- 均衡改善量用綠色（正數）
- 係數改變量若為正數用綠色、負數用灰色
- 若「均衡改善量為 0」顯示為「均衡不變」不顯示數字
- 若「係數改變量為 0」顯示為「係數不變」

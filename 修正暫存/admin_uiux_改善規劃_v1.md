# admin.html UI/UX 改善規劃
> 基於使用情境分析的介面優化方案
> 版本：v1.0 | 建立日期：2026-03-26

---

## 使用情境

- **主要使用者：** Mike（班長），未來可能有助教或學員
- **使用場景：** 上課中即時調整（老師口述，Mike 馬上改）
- **操作規模：** 一次跨部位、跨維度改好幾個地方
- **核心需求：** 速度（沒時間找東西）、不能漏存（上課節奏快）、容錯（改錯要能發現）

---

## 問題清單與優先順序

### P0 — 必修（直接影響資料安全）

**1. 儲存邏輯改為全部存**
- 現狀：儲存只存當前 Tab（規則或問題），切 Tab 後容易漏存
- 改為：「儲存到 Firebase」一次存 rules + questions
- 加「未儲存變更」計數器，顯示有多少筆修改未存

**2. 未儲存提醒強化**
- 現狀：「已修改（未儲存）」只在右上角小灰字
- 改為：有未儲存修改時，儲存按鈕變色（如紅底白字），文字改為「儲存 (3筆修改)」
- 切換 Tab 或關閉頁面時，如有未儲存修改，跳 confirm 提醒

### P1 — 重要（影響操作效率）

**3. 位置感（麵包屑）**
- 現狀：進入編輯後不知道自己在哪
- 改為：頂部 Tab 下方加一行麵包屑
  - 評分規則：`形勢 → 耳 → COUNT min=4`
  - 觀察問題：`眉 → 眉型 → br3 眉形走向`
- 左欄選中項目要有明確的視覺標記（已有 active 樣式，維持）

**4. 操作按鈕常駐顯示**
- 現狀：編輯/刪除/上下按鈕 opacity 0.3，hover 才顯示
- 改為：按鈕始終可見（opacity 0.6），hover 時加深（opacity 1）
- 原因：上課中用觸控設備或快速操作時，隱藏按鈕增加認知負擔

### P2 — 改善（提升可讀性）

**5. 條件樹層級視覺化**
- 現狀：只靠 border-left 和縮排區分層級
- 改為：每層用不同深淺的背景色
  - 第 0 層：白色
  - 第 1 層：淡灰 `rgba(0,0,0,0.02)`
  - 第 2 層：更淡灰 `rgba(0,0,0,0.04)`
- 運算元標籤（COUNT/AND/OR/LR/VETO）加 pill 樣式，更醒目
- 收合長條件樹：超過 3 層的巢狀預設收合，點擊展開

**6. 儲存成功的回饋**
- 現狀：儲存成功只在右上角顯示小字
- 改為：短暫的 toast 通知（綠色底，2 秒後消失），儲存按鈕恢復正常色

### P3 — 加分（未來可做）

**7. 操作歷史（Undo）**
- 每次修改前自動快照，提供「復原上一步」功能
- 上課中改錯了可以快速回退

**8. 修改差異預覽**
- 儲存前顯示「你改了什麼」的摘要
- 避免不小心存入不想要的修改

**9. 鍵盤快捷鍵**
- Ctrl+S / Cmd+S = 儲存
- 上課中最常用的動作應該有快捷鍵

---

## 實作順序建議

1. 先做 P0（儲存邏輯 + 未儲存提醒）— 防止資料遺失
2. 再做 P1（麵包屑 + 按鈕可見度）— 提升操作速度
3. 最後做 P2（視覺層級 + 回饋）— 提升可讀性
4. P3 視需求決定

每個 P 等級可以作為一個 Claude Code 指令批次處理。

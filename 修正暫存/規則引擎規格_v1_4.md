## 八、運算元總覽

| 運算元 | 功能 | 結構 | admin 顯示 |
|--------|------|------|-----------|
| 比對條件 | 觀察答案 = match → true | `{ ref, match, side? }` | `h1 頂骨型相 = 龜背` |
| `AND` | items 全部 true → true | `{ op: "AND", items: [...] }` | 都符合 AND |
| `OR` | items 任一 true → true | `{ op: "OR", items: [...] }` | 或 OR |
| `COUNT` | items 中 true 的加權總數 ≥ min → true | `{ op: "COUNT", min: N, items: [...] }` | 計數 COUNT 最少N項 |
| `LR` | 左右各自判定後匯總 | `{ op: "LR", merge: "any"/"all", each: {...} }` | 左右部位 LR |
| `VETO` | 觸發條件 → 強制判定結果 | `{ veto: [{condition, result}], rule: {...} }` | VETO |
| `NOT` | 條件結果取反 | `{ op: "NOT", item: {...} }` | 不可有 NOT |
| `partResult` | 引用同維度其他部位的結果 | `{ partResult: "部位" / "部位.L" / "部位.R" }` | 引用結果 → 左眉 |
| `group` | 顯示分組（v1.3 新增，v1.4 擴充） | `{ group: "敘述文字", items: [...] }` | 敘述分組 |

所有運算元支援任意巢狀組合。

### group 節點詳細說明（v1.3 新增，v1.4 擴充）

**結構：**
```json
{
  "group": "頂骨要圓",
  "items": [
    { "ref": "h1", "match": "龜背", "side": "L" },
    { "ref": "h1", "match": "龜背", "side": "R" }
  ]
}
```

**用途：** 在 AND/OR/COUNT 的 items 裡把多個條件分組，並加上老師的敘述文字。學員在 13 維度觀察評分頁看到的是摺疊式的分組，點擊敘述展開看具體條件。

**計分邏輯：** group 不參與計分。evaluate 引擎遇到 group 節點時穿透到裡面的 items，每個 item 各自獨立計分（各自有自己的 weight）。

**適用範圍（v1.4 擴充）：**
- group 可出現在 AND、OR、COUNT 的直接子層（v1.3 僅限 COUNT）
- 不能巢狀 group（group 內部的 items 不能再包 group）
- group 內可包含 LR 節點，LR 會正確展開為左右 condResults
- group 和非 group items 可以混合存在
- group 是可選的，沒有 group 的規則完全向下相容

**condResults 處理：** group 內的 items 展開後，每個 condResult item 帶 `groupLabel` 欄位。cpRenderMain 按 groupLabel 分組顯示。

**admin.html 支援（v1.4 完成）：**
- 「+ 新增條件」選單有「敘述分組」選項（AND/OR/COUNT 均可）
- 現有條件旁有「包入分組」按鈕，可選擇包入既有 group 或建立新 group
- group 的敘述文字可編輯
- group 刪除時連同內部 items 一起移除

### NOT 運算元詳細說明（v1.2 新增）

**結構：**
```json
{
  "op": "NOT",
  "item": { "ref": "n5", "match": "無起節" }
}
```

**邏輯：** 遞迴 evaluate item，然後取反（true → false，false → true）。

**注意事項：**
- NOT 只包一個 item（不是 items 陣列）
- item 可以是任何節點類型
- NOT 的結果是布林，不參與 COUNT 的加權計算

## 九、DIMS 與 DIM_RULES 的對應關係

DIMS 陣列定義每個維度的顯示名稱（a/b）和動靜類型（aT/bT）。DIM_RULES 定義 positive/negative。

**關鍵規則：** `DIMS[i].a` 必須等於 `DIM_RULES[i].positive`，`DIMS[i].aT` 必須等於 `DIM_RULES[i].positiveType`。因為 evaluate 引擎算出的 `A` 代表 positive，DIMS 渲染時用 `A` 對應 `a`。

**重要：靜/動標籤必須動態取值，不可寫死陣列。** 正確做法：
```javascript
var colS = d.aT === '靜' ? d.a : d.b;  // 靜的標籤
var colD = d.aT === '動' ? d.a : d.b;  // 動的標籤
```

## 十、Null 防護機制（v1.4 新增）

admin.html 操作（刪除、undo 等）可能在 items 陣列中留下 null 元素。引擎和 condResults 建構程式碼已加入全面防護：

**前台（index_desktop.html）：**
- `evaluate`、`_expandItems`、`_expandLR`、`_collectRefs` 所有 items 遍歷開頭加 `if(!it)return;` 或 `if(!cItem)continue;`
- group 穿透內的子項目遍歷也加 `if(!gi)return;`

**後台（admin.html）：**
- `saveAll` 儲存前遞迴呼叫 `cleanNulls()`，過濾所有 items 陣列中的 null 元素

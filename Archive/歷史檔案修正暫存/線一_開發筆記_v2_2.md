## 技術架構

- 單一 HTML 檔案（HTML + CSS + JS 全部在同一檔案）
- Firebase Firestore 儲存評分資料
- Anthropic Claude API 生成 AI 評析（AI 評析目前已從介面移除，函數保留備用）
- 無帳號系統，用姓名識別使用者
- **兩個 `<script>` 區塊**：第一個是 `(function(){...})()` 包住的知識頁邏輯（含 `KD` 陣列），第二個是主程式。`KD` 在主程式 scope **不可見**。

### 7b condResults 自動產生

- `_buildPartCond()` 從 JSON 規則樹自動產生 condResults
- `_expandLR()` 展開 LR 節點為左右各一組 items
- `_expandItems()` 展開 COUNT/AND/OR 的 items，支援 group 穿透（含 group 內 LR 展開）
- `_flattenLeaf()` ref 葉節點 → condResult item（含 label/ok/ids/side/val/wt/connType/groupLabel）
- `_flattenCompound()` 複合節點合成一行（含 connType 區分 AND/OR）
- `_makeThreshold()` 自動產生門檻描述文字
- `_nodeLabel()` 節點描述文字（AND→以及、OR→或、NOT→不可以有）
- `_collectRefs()` 收集所有 ref ID（支援 group 穿透）
- label 由規則樹自動產生，不再手動維護
- partResult 引用顯示為「左眉達標」「鼻達標」等
- null item 防護：所有 forEach 回調開頭加 null 檢查（v2.2）

### group 敘述分組（v2.1 新增，v2.2 擴充）

- AND/OR/COUNT items 均支援 `{ group: "敘述文字", items: [...] }` 分組節點（v2.2 擴充到 AND/OR）

## 重要技術注意事項

- 兩個 `<script>` 區塊：第一個含 KD 陣列（知識頁），KD 在主程式 scope 不可見
- `condResults[dimIdx][partIdx]` 由 `_buildPartCond()` 從 JSON 規則樹自動產生；注意 `condResults[dimIdx]` 是 **object**（不是 array），遍歷需用 `Object.keys()`
- DIMS 陣列的 a/b 順序必須跟 DIM_RULES 的 positive/negative 一致（A=positive）
- **靜/動標籤不可寫死陣列**：必須從 `DIMS[i].aT` 動態判斷哪個是靜、哪個是動（圓方/攻守/奇正/進退的 a 是動不是靜）
- CSS 變數 `--static:#7A9E7E`、`--active:#C17A5A` 統一全站配色
- `cur`（13維度觀察評分頁）和 `cpCur`（敏感度分析頁）同步：dimGoto 和 cpGoto 都同時設定兩者
- group 節點可出現在 AND/OR/COUNT 的直接子層，evaluate 和 _expandItems 均穿透處理
- LR 內的 ref 節點不應帶 side 屬性（由 LR 外層傳入）；`_flattenLeaf` 遇到 `side:null` 會 fallback 到外層 side
- 所有 items 遍歷都有 null 防護（`if(!it)return;`）

## 產品定位

人相兵法不是算命工具，是一套**觀察與分析框架**。它以面相學、氣色、體相為原始素材，結合先天/後天/運氣的兵法結構，提供一個可以持續演進的評析系統。

這個平台的本質是：**面相學知識的創作、研究、和累積平台**。

- 不是「輸入資料、輸出答案」的系統
- 而是「指引框架的載體」——框架本身可以被設計、調整、演進
- 核心骨架（13個維度及其意義）永遠不變
- 觀察內容、評分標準、指引解釋都可以演進

## 使用者

### 三層使用者

**老師**
- 課程內容的原始來源
- 不用電腦，只口述
- 透過班長（Mike）與系統互動
- 收報告方式：LINE

**班長（Mike）**
- 管理官方模組，發布每年課程版本
- 設計報告格式
- 維護系統

**學員**
- 使用官方模組練習
- 建立個人版本模組
- 點對點分享模組
- 上台分享真實情況與評析對照

### 使用規模
- 現在：10幾人
- 未來上限：約 100 人

## 核心功能

### 1. 帳號系統
- Email + 密碼登入
- 三種角色：班長、學員（未來可加老師）
- 班長可管理學員帳號

### 2. 模組系統

**模組是什麼**
把「觀察題目」和「評分標準」打包成一個版本，例如「2026 課程模組」。

**模組的三個層次**
- **官方模組**：由班長發布，對應每年課程內容
- **個人模組**：學員複製官方模組後自行調整，包括修改條件甚至重新設計
- **分享模組**：學員點對點分享給特定人，收到的人可以匯入為自己的獨立複製品

**模組版本管理**
- 舊版本保留，不覆蓋
- 新舊版本可以並排比較
- 每個模組有名稱（使用者自訂）和建立日期

**模組包含的內容**
- 觀察題目清單（每題的問題、選項、說明文字）
- 13個維度的評分規則（每個部位對每個維度的判斷條件）
- 判斷邏輯：AND / OR / NOT 條件組合

### 3. 評析流程

**標準流程**
1. 選擇使用哪個模組
2. 填寫觀察問題
3. 系統自動計算 13 維度動靜和係數
4. 有需要時，主動請 AI 生成評析（AI 不主動介入）

**AI 評析流程**
- 學員主動觸發
- AI 生成初步評析
- AI 可以詢問學員問題以提升評析準確性（雙向對話）
- 學員回答後，AI 調整評析方向
- 生成最終評析報告

**現階段 AI 原則**
- AI 不在填寫過程中給提示或預測
- 只在學員主動請求時才介入
- 目的是輔助理解，不是替代學習

### 4. 報告系統
- 生成可分享的報告
- 適合 LINE 傳送的格式（圖片為主）
- 班長可設計報告格式

### 5. 課堂案例累積

**學習回饋**
- 學員事後可填入真實情況對照（我評析出 X，實際上 Y）
- 當事人明確同意後，這筆資料才能被記錄為訓練案例

**完整案例結構**
每一筆高品質案例包含：
- 觀察資料（使用哪個模組、每題的答案）
- 評析結果（13維度動靜和係數）
- AI 對話記錄（AI 問了什麼、學員怎麼回答）
- 真實對照（學員事後填的實際情況）
- 老師驗證（班長標記資料品質）

**長期目標**
累積足夠的高品質案例後，訓練專屬的面相學 AI 模型。

## 技術架構

### 前端
- **框架：** React
- **響應式：** 桌機版 / 平板版 / 手機版（同一套程式碼）
- **離線支援：** Offline-first，本地儲存資料，連線後自動同步

### 後端
- **帳號：** Firebase Authentication（Email/密碼）
- **資料庫：** Firebase Firestore
- **離線同步：** Firestore offline persistence
- **AI Proxy：** Firebase Cloud Functions（API key 存在後端，學員不需要自己申請）
- **部署：** Firebase Hosting

### AI
- **現在：** Claude API（透過 Cloud Functions proxy）
- **未來：** 累積資料後訓練專屬模型

## 資料架構（草稿）

### Collections

```
users/
  {userId}/
    name: string
    email: string
    role: 'admin' | 'student'
    createdAt: timestamp

modules/
  {moduleId}/
    name: string
    authorId: string
    baseModuleId: string | null  // 從哪個模組複製來的
    createdAt: timestamp
    isOfficial: boolean
    questions: [...]  // 觀察題目清單
    rules: {...}      // 13維度評分規則

analyses/
  {analysisId}/
    userId: string
    moduleId: string
    subjectName: string
    observations: {...}  // 每題的答案
    results: {...}       // 13維度計算結果
    aiDialogue: [...]    // AI 對話記錄
    realityCheck: string // 真實情況對照
    isTrainingData: boolean  // 是否同意作為訓練資料
    qualityMark: string | null  // 班長驗證標記
    createdAt: timestamp
```

## 開發階段規劃

### 第一階段：基礎建設
- 帳號系統（Firebase Auth）
- 資料架構實作（Firestore）
- Offline-first 機制
- 響應式 UI 框架（React）

### 第二階段：核心功能
- 模組系統（建立、複製、版本管理）
- 觀察填寫介面（從線一移植）
- 13維度計算引擎（規則引擎取代硬寫邏輯）
- AI 評析（透過 Cloud Functions）
- 報告生成和分享

### 第三階段：進階功能
- 點對點模組分享和匯入
- 模組比較
- 課堂案例記錄和同意機制
- 資料累積整理

## 從線一移植的內容

線一的以下功能需要移植到線二：
- 觀察問題題目和 UI 設計

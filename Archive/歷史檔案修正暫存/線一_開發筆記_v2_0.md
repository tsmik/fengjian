## 技術架構

- 單一 HTML 檔案（HTML + CSS + JS 全部在同一檔案）
- Firebase Firestore 儲存評分資料
- Anthropic Claude API 生成 AI 評析（AI 評析目前已從介面移除，函數保留備用）
- 無帳號系統，用姓名識別使用者
- **兩個 `<script>` 區塊**：第一個是 `(function(){...})()` 包住的知識頁邏輯（含 `KD` 陣列），第二個是主程式。`KD` 在主程式 scope **不可見**。

### 7b condResults 自動產生（v2.0 新增）

- `_buildPartCond()` 從 JSON 規則樹自動產生 condResults
- `_expandLR()` 展開 LR 節點為左右各一組 items
- `_expandItems()` 展開 COUNT/AND/OR 的 items
- `_flattenLeaf()` ref 葉節點 → condResult item（含 label/ok/ids/side/val/wt）
- `_flattenCompound()` 複合節點合成一行
- `_makeThreshold()` 自動產生門檻描述文字
- `_nodeLabel()` 節點描述文字
- `_collectRefs()` 收集所有 ref ID
- label 由規則樹自動產生（格式：「左頂骨型相（龜背）」），不再手動維護
- partResult 引用顯示為「左眉達標」「鼻達標」等
- commit cb58881（初版，淨減 1005 行）+ bugfix（partResult 帶 dot 的引用修正）

## admin.html — 管理介面

- **網址：** https://tsmik.github.io/fengjian/admin.html
- **Firebase 存取：** 讀寫 `settings/rules` 和 `settings/questions`

### admin.html — 評分規則編輯器

- 左欄：13 個維度列表（按先天/運氣/後天分類，維度色塊標記）
- 中欄：9 個部位的規則樹，可展開/收合
- 部位 header 顯示條件數量（如「頭 · 8個條件」）
- 展開/收合箭頭狀態（▼/▲）
- 支援 7 種運算元：AND（都符合）、OR（或）、COUNT（計數）、LR（左右部位）、VETO、NOT（不可有）、partResult（引用結果）
- 運算元 pill 標籤：中文+英文、顏色區分（AND/OR/COUNT 紅色、LR 金色、NOT 紫色、partResult 灰色）
- COUNT 顯示「最少N項 min=N」pill
- LR 顯示「都符合」或「一邊符合即可」
- ref 節點顯示：左右前綴（左/右）、match 值 + hint tooltip + 預覽文字
- 編輯表單：ref 下拉按部位過濾、選項含 hint 說明、Side/Weight 收在進階設定
- 新增條件：選擇節點類型（比對條件/AND/OR/COUNT/NOT）
- 運算元類型切換（AND↔OR↔COUNT）
- 根層級新增條件（自動包進容器）
- LR 內新增條件（自動包進容器）
- 層級背景色 + border-left 加深
- 條件按部位過濾題目（中停/下停逐步選擇）
- partResult 提醒

### admin.html — 觀察問題編輯器

- 左欄：11 個觀察部位列表，顯示各部位題數
- 中欄：按 section 分組顯示題目
- 編輯：text / paired / 選項（v + hint），新增/刪除選項
- 刪除題目：自動掃描規則 JSON，有引用則警告
- 新增題目：自動產生下一個 ID
- 上下移動、Section 標題雙擊可改名

### admin.html — 通用功能（v1.9 新增）

- **P0 儲存邏輯：** 一次同時存 rules + questions，不再分 Tab 存
- **P0 未儲存提醒：** dirty 計數器、儲存按鈕紅底脈動、beforeunload 警告、Tab 切換提醒
- **P1 麵包屑：** 顯示當前位置（評分規則 → 維度 → 部位）
- **P1 按鈕常駐：** 編輯/刪除按鈕預設 opacity 0.6
- **P2 層級視覺化：** 深度背景色 + border-left 顏色遞增
- **P2 Toast 通知：** 儲存成功綠色 / 失敗紅色，2 秒消失
- **P3 Undo：** 最多 30 步，Cmd+Z 快捷鍵，儲存後清空
- **P3 鍵盤快捷鍵：** Cmd+S 儲存、Cmd+Z 復原
- **P3 差異預覽：** 儲存前顯示具體修改內容（題目欄位級別、規則條件數變化）
- **刪除確認：** 顯示影響範圍（包含 N 個條件）

### admin.html — 目前限制

- 無權限控管（任何人知道網址都能改）
- TH 門檻編輯已移除（步驟 7a 完成）

## index_desktop.html — Firebase 規則讀取

- **啟動流程：** `startApp()` 登入後，先從 Firebase `settings/rules` 讀取 `rulesJson`，成功則覆蓋 `DIM_RULES`，失敗則 fallback 到內建的 `DIM_RULES_DEFAULT`
- **變數：** `DIM_RULES_DEFAULT`（內建 fallback）、`DIM_RULES`（實際使用）、`_rulesSource`（追蹤來源，'Firebase' 或 '內建'）
- **與 admin.html 同步：** admin.html 儲存規則到 Firebase → 下次任何人登入 index_desktop.html 即讀到最新規則
